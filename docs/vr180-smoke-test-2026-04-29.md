## Smoke test 절차 + 결과

### 사용 단말 / 콘텐츠

- 단말: Galaxy XR (R34YA0007ZJ, SM-I610), 알파13 SDK
- 콘텐츠: 사용자 보유 4K SBS 영상 1편 (`/sdcard/동영상/DSVR-995 ... [VR] A._sbs.mp4`, 4.48GB)
  - codec: H.264 High@L5.2, 4096×2048, 60fps, 30Mbps
  - 메타: sv3d + st3d + equi box 포함 (equirectangular VR180)
- (참고) 초기에 외부 샘플 (Internet Archive `deathsinger-up-close-3-d-vr-180`, 6.5MB) 도 시도 — 4K H.264 같은 codec mismatch 로 폐기, 사용자 실콘텐츠로 대체.

### Step-by-step 결과

| 시도 | 변경 | 결과 |
|---|---|---|
| 1 | baseline (변경 없음) | `AMediaCodec.configure failed` — codec reject |
| 2 | `SURFACE_PIXEL_DIM` 1920×1080 → 4096×2048 | 같은 실패 — surface 크기 무관 |
| 3 | VLC option `--codec=mediacodec_jni,**none** → mediacodec_jni,**avcodec**` (SW fallback 허용) | SW decode 작동, 끊김/macroblock — 4K@60fps SW decode 부담 |
| 4 | `SCREEN_SHAPE` Quad → `Hemisphere(1.0f)` | hemisphere 그려지나 작은 dome 으로 정면에 보임 |
| 5 | `SCREEN_POSE` `(0, 0.3, -3)` → `(0, 0, 0)` (origin) | dome 가까워지지만 여전히 정면 객체 |
| 6 | `Hemisphere(10.0f)` → `Hemisphere(20.0f)` → `Hemisphere(30.0f)` → `Hemisphere(40.0f)` (사용자 직접 조정 — 30 도 가장자리 cover 부족, 40 으로 확정) | wrap-around immersion 확인 ✓ |
| 7 | `--no-mediacodec-dr` 제거 (DR 활성) + 코덱 list `mediacodec_jni,avcodec` | **HW decode 성공** — 4K 끊김 없이 재생, c2 params reject 해결 |

### Smoke test 핵심 발견

1. **Galaxy XR + Jetpack XR alpha13 의 `SurfaceEntity.Shape.Hemisphere(radius)` + `StereoMode.SIDE_BY_SIDE` 가 equirectangular VR180 을 정상 매핑**. design doc Open Q #1 (projection 호환성) 해소 — 코드 작업 진행 가치 입증.
2. **`--no-mediacodec-dr` 가 c2 (codec2 modern stack) configure 를 거부시키는 진짜 원인**. 4K H.264 High@L5.2 가 codec capability 표 (`c2.qti.avc.decoder` 가 8K 까지 지원) 와 일치하지만 DR 없이 configure 실패. DR 활성화로 해결.
3. **SURFACE_PIXEL_DIM 은 codec 출력 frame 과 정확히 일치해야 한다**. 더 크게 잡으면 codec 가 빈 영역 포함 surface 에 native frame 만 좌상단에 쓰고 hemisphere/Quad 가 빈 영역까지 매핑 → 영상 작아지고 위치 어긋남 (8K dim 시도로 검증).
4. **VR180 hemisphere immersion 의 적정 radius ≈ 30m**, pose = origin. 1m 은 머리 주변 작은 dome, 10m 은 wrap-around 시작이지만 가까운 느낌, 20~30m 부터 멀리 있는 immersion 감.

## Step 1+2+3-lite 구현

Smoke test 검증치를 SBS_PANEL 회귀 없이 코드로 정착.

### 1. `xr/SpatialMode.kt` (신규)

```kotlin
enum class SpatialMode {
    NONE,                  // XR 비대상
    SBS_PANEL,             // Quad + SIDE_BY_SIDE — 기존 동작
    VR180_HEMISPHERE,      // Hemisphere + SIDE_BY_SIDE — VR180 immersion
}
```

향후 확장 자리 (주석): MONO_360_SPHERE, TOP_BOTTOM_360_SPHERE.

### 2. `xr/XrConfig.kt` 리팩토링

단일 `SCREEN_POSE` / `SCREEN_SHAPE` / `SURFACE_PIXEL_DIM` 상수 → mode 별 helper 함수.

| Mode | Pose | Shape | Surface dim |
|---|---|---|---|
| `SBS_PANEL` | `(0, 0.3, -3.0)` | `Quad(6.4 × 3.6)` | `1920×1080` |
| `VR180_HEMISPHERE` | `(0, 0, 0)` | `Hemisphere(40.0f)` | `4096×2048` |

추가 helper:
- `vr180PatternMatch(name)` — `[vr]` / `vr180` / `_180_` 등 키워드 검사
- `detectSpatialMode(name)` — SBS keyword 없음 → NONE / SBS+VR180 → HEMISPHERE / SBS only → PANEL
- `screenPose(mode)` / `screenShape(mode)` / `surfacePixelDim(mode)` — mode → 파라미터

### 3. `xr/XrSurfaceController.kt`

`setupStereoSurface(mp)` → `setupStereoSurface(mp, mode: SpatialMode)`.
- `mode=NONE` → 즉시 false (호출자 fallback)
- `mode=VR180_HEMISPHERE` → Movable/Resizable 부착 skip (resize callback 이 Quad 로 덮어써 hemisphere 깨짐)
- `applyVideoAspect()` → `currentMode == SBS_PANEL` 일 때만 동작 (hemisphere 는 모양 고정)

### 4. `SbsPlayerActivity.java`

```java
@Override
protected boolean attemptStereoTakeover(MediaPlayer mp, String sourceName) {
    if (xr == null) return super.attemptStereoTakeover(mp, sourceName);
    SpatialMode mode = XrConfig.detectSpatialMode(sourceName);
    if (mode == SpatialMode.NONE) return super.attemptStereoTakeover(mp, sourceName);
    if (!xr.setupStereoSurface(mp, mode)) return super.attemptStereoTakeover(mp, sourceName);
    ...
}
```

VLC option 변경:
- 이전: `--codec=mediacodec_jni,none` + `--no-mediacodec-dr`
- 현재: `--codec=mediacodec_jni,avcodec` (DR 기본값 = 활성, no-DR flag 제거)

## VR180 / SBS_PANEL 구분 로직 (현재 = 파일명 heuristic)

**핵심 — 현재 구분은 파일명 키워드 검사만. MP4 메타데이터 parsing 은 미구현 (TODO-VR180-3).**

### `XrConfig.detectSpatialMode(name)` 흐름 (2026-04-29 갱신)

```
1. VR180 keyword 있나?
   VR180_FILENAME_KEYWORDS = [vr180, vr_180, vr-180, [vr], [180], _180_]
   → 있음: VR180_HEMISPHERE  (SBS keyword 없어도 매핑됨)

2. SBS keyword 있나?
   SBS_FILENAME_KEYWORDS = [_sbs, _3d, .sbs., [sbs], [3d]]
   → 있음: SBS_PANEL

3. 둘 다 없음 → NONE (XR 분기 진입 X, MainActivity 일반 path)
```

소문자 변환 후 `contains` 검사. **우선순위: VR180 먼저 → SBS 다음 → NONE**.

(초기 구현은 SBS keyword 가 prerequisite 였으나 사용자 요청으로 완화 — VR180 keyword 만 있어도 hemisphere immersion 진입 가능.)

라우팅 영향: `FileListActivity.launchPlayer` 도 `sbsPatternMatch` 단독 검사 → `detectSpatialMode != NONE` 으로 변경. VR180 keyword 만 있는 파일도 SbsPlayerActivity 로 라우팅 + Bundle launch (Full Space).

## VR180-2 구현 — SURFACE_PIXEL_DIM 동적 매칭 (2026-04-29 후속)

mode 별 hard-coded surface dim 이 영상 실제 frame 크기와 mismatch 시 영상 작아지고 위치 어긋남 (smoke test 시도 2 / 8K dim 실험으로 확인). pre-probe 로 실제 frame 크기 알아내 surface dim 을 정확히 매칭하도록 변경.

### 변경 내역

**`XrConfig.surfacePixelDim(mode, videoW, videoH)`** — 두 param 추가. `videoW > 0 && videoH > 0` 면 그 값, 0 이면 mode 별 default fallback (`SBS_PANEL` → 1920×1080, `VR180_HEMISPHERE` → 4096×2048).

**`XrSurfaceController.setupStereoSurface(mp, mode, videoW, videoH)`** — videoW/H 추가 default param (Kotlin), Java 호출자는 두 param 명시.

**`SbsPlayerActivity.attemptStereoTakeover`** — `MediaMetadataRetriever` 로 `getIntent().getData()` URI 의 `METADATA_KEY_VIDEO_WIDTH/HEIGHT` 사전 probe. ~5ms 오버헤드, 실패 시 `[0, 0]` 반환 (mode default fallback).

```java
private int[] probeVideoDimensions() {
    Uri uri = getIntent() != null ? getIntent().getData() : null;
    if (uri == null) return new int[] { 0, 0 };
    MediaMetadataRetriever mmr = new MediaMetadataRetriever();
    try {
        mmr.setDataSource(this, uri);
        String w = mmr.extractMetadata(MediaMetadataRetriever.METADATA_KEY_VIDEO_WIDTH);
        String h = mmr.extractMetadata(MediaMetadataRetriever.METADATA_KEY_VIDEO_HEIGHT);
        return new int[] { w != null ? Integer.parseInt(w) : 0, h != null ? Integer.parseInt(h) : 0 };
    } catch (Exception e) {
        Log.w(TAG, "MediaMetadataRetriever probe 실패: " + e);
        return new int[] { 0, 0 };
    } finally {
        try { mmr.release(); } catch (Exception ignored) {}
    }
}
```
